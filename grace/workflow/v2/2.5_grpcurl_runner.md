# Grpcurl Runner (v2)

You validate the connector's freshly-implemented items by starting the gRPC service once, running `grpcurl` against each item's payload template, capturing pass/fail + response, and writing a per-connector results JSON. You do NOT modify code, do NOT commit.

You are spawned **once per connector** by the Connector Session (`2_connector.md`) at the end of Phase 4, after all per-item codegen has finished.

---

## Inputs

| Variable | Description | Example |
|---|---|---|
| `{CONNECTOR}` | Connector slug (lowercase) | `stripe` |
| `{ITEMS}` | JSON array of implemented items (with `id`, `name`, `kind`, `source_flow`, `grpcurl_payload` path) | `[{...}, ...]` |
| `{PAYLOADS_DIR}` | Directory holding per-item payload templates | `grace/workflow/v2/work/stripe_payloads` |
| `{OUTPUT}` | Path to the per-connector results JSON to update | `grace/workflow/v2/work/stripe_results.json` |

---

## Rules

1. **One service start per connector**, not per item. Starting/stopping `make run` between items would multiply runtime. Start once, hammer the running service across all items, stop once.
2. **No retry without code change**. If grpcurl fails on an item, record the failure and proceed to the next item — do NOT re-invoke. (The codegen agent has already done its retry budget.)
3. **No edits to creds.json**. Read-only.
4. **Sequential within the connector**. Even if you wanted to parallelize, the service binds a single port; concurrent grpcurl calls are fine but errors compound — keep it serial for clarity.
5. **Always stop the service**, even on partial failure. Use `trap` to ensure cleanup.

---

## Phase 1: Pre-flight

```bash
test -f Makefile || { echo "no Makefile"; exit 1; }
test -f creds.json || { echo "no creds.json"; exit 1; }
test -d "{PAYLOADS_DIR}" || { echo "no payloads dir"; exit 1; }
which grpcurl >/dev/null || { echo "grpcurl not installed"; exit 1; }
jq -e --arg c "{CONNECTOR}" '.[$c] // .[($c | ascii_upcase)] // .[($c | ascii_downcase)]' creds.json >/dev/null \
    || { echo "no creds for {CONNECTOR}"; exit 1; }
```

If any check fails: write a stub `{OUTPUT}` with `status: "FAILED"`, set every item's `grpcurl_status` to `SKIPPED` with reason `"runner_preflight_failed"`, and return.

---

## Phase 2: Start the Service

```bash
LOG=/tmp/grace-v2-{CONNECTOR}-service.log
PORT="${UCS_GRPC_PORT:-8080}"

# Background start. Use `make run-quiet` if available; else `make run`.
( make run > "$LOG" 2>&1 ) &
SERVICE_PID=$!
trap "kill $SERVICE_PID 2>/dev/null; wait $SERVICE_PID 2>/dev/null; true" EXIT

# Wait for readiness — try grpcurl list against the configured port up to 60s.
for i in $(seq 1 30); do
  if grpcurl -plaintext "localhost:$PORT" list >/dev/null 2>&1; then
    READY=1; break
  fi
  sleep 2
done
[[ "${READY:-0}" == 1 ]] || {
  tail -50 "$LOG" >&2
  echo "service did not become ready on :$PORT"; exit 1;
}
```

If service does not become ready: write `{OUTPUT}` with every item's `grpcurl_status: SKIPPED` reason `"service_unhealthy"`. Return FAILED.

---

## Phase 3: Per-Item grpcurl

Initialize `RESULTS_ITEMS=[]`. For each item in `{ITEMS}` (sequential):

```bash
PAYLOAD_FILE="{PAYLOADS_DIR}/${item.id}.json"
[[ -f "$PAYLOAD_FILE" ]] || { record SKIPPED reason="payload_missing"; continue; }

GRPC_SERVICE=$(jq -r '.grpc_service' "$PAYLOAD_FILE")
GRPC_METHOD=$(jq -r '.grpc_method'  "$PAYLOAD_FILE")
PAYLOAD=$(jq '.payload' "$PAYLOAD_FILE")

# Substitute creds: replace literal "<auth-header-template — runner substitutes from creds.json>"
# with the connector's auth header value derived from creds.json.
AUTH_VALUE=$(jq -r --arg c "{CONNECTOR}" '
    (.[$c] // .[($c | ascii_upcase)] // .[($c | ascii_downcase)])
    | (.api_key // .secret_key // .auth_token // .key // "")
' creds.json)
META_FLAGS=$(jq -r --arg auth "$AUTH_VALUE" '
    .metadata
    | to_entries
    | map("-rpc-header \(.key)=\(if .value | tostring | contains("<auth-") then $auth else .value end)")
    | join(" ")
' "$PAYLOAD_FILE")

# Build the request body
REQ_BODY=$(echo "$PAYLOAD" | jq -c '.')

# Run grpcurl
RESPONSE_FILE=$(mktemp)
START_TIME=$(date +%s%3N)
if grpcurl -plaintext -d "$REQ_BODY" $META_FLAGS \
       "localhost:$PORT" \
       "$GRPC_SERVICE/$GRPC_METHOD" \
     > "$RESPONSE_FILE" 2>&1
then
    DURATION_MS=$(( $(date +%s%3N) - START_TIME ))
    RESPONSE=$(cat "$RESPONSE_FILE")
    # Decide pass/fail using expected_status_classes
    EXPECTED=$(jq -r '.expected_status_classes | join("|")' "$PAYLOAD_FILE")
    if echo "$RESPONSE" | grep -qE "\"status\":\\s*\"(${EXPECTED})\"" \
       || echo "$RESPONSE" | grep -qE "\"(authorized|charged|pending|requires_customer_action)\"" -i; then
        STATUS="SUCCESS"; REASON=""
    else
        STATUS="FAILED"; REASON="status_class_mismatch"
    fi
else
    DURATION_MS=$(( $(date +%s%3N) - START_TIME ))
    RESPONSE=$(cat "$RESPONSE_FILE")
    STATUS="FAILED"
    REASON=$(echo "$RESPONSE" | head -3 | tr '\n' ' ' | head -c 200)
fi
rm -f "$RESPONSE_FILE"

# Append to RESULTS_ITEMS
RESULTS_ITEMS=$(jq --argjson item '<item>' \
    --arg status "$STATUS" \
    --arg reason "$REASON" \
    --arg response "$RESPONSE" \
    --argjson duration "$DURATION_MS" \
    '. + [{
        id: $item.id,
        name: $item.name,
        kind: $item.kind,
        source_flow: $item.source_flow,
        grpcurl_status: $status,
        reason: $reason,
        duration_ms: $duration,
        response_preview: ($response | .[0:2000])
    }]' <<<"$RESULTS_ITEMS")
```

Note: cap `response_preview` at 2000 chars so the results JSON stays small.

---

## Phase 4: Stop the Service

The `trap` from Phase 2 handles this on exit. To stop early after all items:

```bash
kill "$SERVICE_PID" 2>/dev/null
wait "$SERVICE_PID" 2>/dev/null || true
trap - EXIT
```

---

## Phase 5: Write Results JSON

Update `{OUTPUT}`. If the file already exists (Connector Session may have pre-written a stub with codegen statuses), merge by item id:

```bash
if [[ -f "{OUTPUT}" ]]; then
  jq --argjson new "$RESULTS_ITEMS" '
    .items = (
      .items as $existing |
      $new as $newitems |
      ($existing | map(. as $e | $newitems[]? | select(.id == $e.id) // $e))
    )
  ' "{OUTPUT}" > "{OUTPUT}.tmp" && mv "{OUTPUT}.tmp" "{OUTPUT}"
else
  jq -n --arg c "{CONNECTOR}" --argjson items "$RESULTS_ITEMS" '
    {connector: $c, items: $items}
  ' > "{OUTPUT}"
fi
```

The merge keeps prior fields like `codegen_status` while adding `grpcurl_status` / `reason` / `duration_ms` / `response_preview`.

---

## Phase 6: Return

Print a one-line summary:

```
GRPCURL: {CONNECTOR} — total={N} success={N_success} failed={N_failed} skipped={N_skipped} | output={OUTPUT}
```

On any phase 1-2 hard failure (service won't start, no creds): exit code 0 still — the Connector Session needs the results JSON written. Mark items SKIPPED with the appropriate reason.

---

## Notes

- The connector-specific auth header key (`api_key` vs `secret_key` vs `auth_token`) varies. The jq fallback chain in Phase 3 covers common shapes; for connectors with non-standard creds keys, the codegen agent should put a more specific placeholder in the payload template's `metadata`.
- `expected_status_classes` is a hint; the regex match is forgiving (mixed case, several variants). Tighten per connector as patterns emerge.
- If a connector requires a non-default port (some local dev setups), set `UCS_GRPC_PORT` before invoking the orchestrator. The runner inherits it.
- `response_preview` is capped to keep the results JSON committable. The full response goes to `/tmp/grace-v2-{CONNECTOR}-{item.id}.response` for ad-hoc debugging during the run.
