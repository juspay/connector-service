# Unified Connector Service (UCS) Metadata Usage Guide

This document deterministically lists the expected JSON structures and purposes for the three metadata fields used across all connectors in the UCS.

---

## 1. Account Configuration Fields
**Mapping:** Proto `merchant_account_metadata` (Id 33) $\rightarrow$ Domain `connector_meta_data` (with underscore).  
**Purpose:** Stores static credentials, routing IDs, or environment-specific configuration provided by the merchant.

| Connector | Expected JSON Structure | Purpose |
| :--- | :--- | :--- |
| **Adyen** | `{"endpoint_prefix": "string"}` | Constructs live API URLs. |
| **Braintree** | `{"merchant_account_id": "string", "merchant_config_currency": "enum"}` | Routes to specific sub-merchant accounts. |
| **Cybersource**| `{"disable_avs": bool, "disable_cvn": bool}` | Controls fraud check behavior. |
| **Gigadat** | `{"site": "string"}` | Merchant site identifier for Interac. |
| **Truelayer** | `{"merchant_account_id": "string", "account_holder_name": "string", "private_key": "string", "kid": "string"}` | Credentials for Open Banking signing. |
| **WorldpayVantiv** | `{"report_group": "string"}` | Groups transactions in the Vantiv dashboard. |

---

## 2. Transaction State Fields
**Mapping:** Proto `connector_metadata` (Id 41) $\rightarrow$ Domain `connector_metadata` (no underscore).  
**Purpose:** Stores dynamic transaction state (IDs, tokens, flow flags) generated by the connector and required for subsequent steps (Capture, Sync, Refund).

| Connector | Expected JSON Structure | Purpose |
| :--- | :--- | :--- |
| **Paypal** | `{"authorize_id": "string", "capture_id": "string", "psync_flow": "enum", "order_id": "string"}` | Tracks PayPal resource IDs across flows. |
| **Checkout** | `{"psync_flow": "Standard | Simplified"}` | Determines the synchronization strategy. |
| **Worldpay** | `{"link_data": "string"}` | Maintains state during 3DS authentication. |
| **NexixPay** | `{"three_d_s_auth_result": {...}, "payment_id": "string"}` | Stores authentication results for validation. |
| **Paysafe** | `{"payment_handle_token": "string"}` | Temporary token for completing payments. |
| **Revolv3** | `{"order_id": "string"}` | Internal order ID for synchronization. |

---

## 3. General Metadata Fields
**Mapping:** Proto `metadata` (Id 16) $\rightarrow$ Domain `metadata`.  
**Purpose:** Stores arbitrary merchant-provided key-value pairs for reporting, risk analysis, or passthrough to the connector dashboard.

| Connector | Expected JSON Structure | Purpose |
| :--- | :--- | :--- |
| **Stripe** | `{"order_id": "string", "customer_info": "string", ...}` | Passed directly to Stripe's metadata map. |
| **Adyen** | `{"device_fingerprint": "string", "store": "string", "platform_chargeback_logic": {...}}` | Risk analysis and fraud prevention data. |
| **Stax** | `{"tax": 100}` | Specific reporting for transaction tax. |
| **Checkout** | `{"any_key": "any_value"}` | Passthrough for merchant-defined reporting logic. |
| **Billwerk** | `{"external_id": "string"}` | External reference for accounting reconciliation. |

---

## **Critical Mapping Summary**

| Data Type | Proto Field (Input) | Domain Field (Internal) | Lifetime |
| :--- | :--- | :--- | :--- |
| **Account Config** | `merchant_account_metadata` | `connector_meta_data` | Permanent (Setup) |
| **Transaction State** | `connector_metadata` | `connector_metadata` | Per-Payment (Dynamic) |
| **Business Data** | `metadata` | `metadata` | Per-Payment (Passthrough) |

**Note:** In the current repository, there is a known mapping issue in `backend/domain_types/src/types.rs` where some request conversions (Authorize, Capture) mistakenly ignore the `connector_metadata` input.
